# Effectuer une recherche

## 1. Préciser le besoin d'information

### Définir et justifier un sujet de recherche
Un sujet de recherche peut être justifié par divers contextes :

* **Entreprise** : Développement d'un nouveau produit répondant à un besoin spécifique.
* **Académique** : Poursuite d'études aux cycles supérieurs (maîtrise ou doctorat).
* **Personnel** : Passion pour un sujet ou réalisation d'un projet technique (ex: fabriquer un drone, créer un jeu 3/4).
* **Social** : Recherche sur les méthodes d'influence (ex: analyse des mécanismes du subconscient pour un parti politique).

### Définir les éléments du sujet
Il est essentiel de discerner les points d'ancrage de la recherche :

* Associer des **mots-clés** précis au sujet.
* Utiliser des outils visuels comme une **carte mentale** pour organiser les idées.

### Analyser les besoins en information
Une fois les éléments identifiés, il faut épurer le sujet :

* Ne conserver que les besoins essentiels directement liés à la recherche.
* Documenter les raisons pour lesquelles certains éléments sont écartés afin de garder une trace du processus.

### Produire un plan provisoire
Le plan provisoire est une structure simplifiée (liste à puces) comprenant :

* Page de présentation et table des matières.
* **Introduction** : Sujet amené, sujet posé et sujet divisé (prévoir une liaison avec le projet).
* **Développement** : 
    * Besoins de la recherche.
    * Historique (limité à l'essentiel).
    * Méthodologie.
    * Résultats obtenus (comparaison avec les attentes, efforts fournis).
    * Débat (analyse des méthodes alternatives).
    * Sujets similaires.
* **Conclusion** : Rappel, remise en question et pistes futures.
* **Médiagraphie**.

## 2. Rechercher l’information

### Critères de sélection des sources
* Les critères doivent être établis de manière **objective**, sans tenir compte des opinions personnelles.
* Le rang dans un moteur de recherche populaire ne constitue pas un critère de fiabilité.
* Privilégier les institutions spécialisées en recherche.
* Se méfier des sources provenant d'entreprises privées pouvant avoir des intérêts commerciaux ou des biais.
* **Important (2026)** : Les LLM (ChatGPT, Claude, Gemini) ne doivent PAS être utilisés comme sources primaires ou secondaires dans un rapport académique, mais plutôt comme outils d'aide à la réflexion et à l'organisation.

### Fiabilité des sources (Le cas Wikipédia)
* Pour les **sciences exactes** (ex: mathématiques), Wikipédia est généralement fiable.
* Pour les sujets "flexibles" (histoire, politique), les informations peuvent varier selon la langue et le point de vue national (ex: les divergences entre les versions russe, ukrainienne et allemande concernant la Crimée).
* **Astuce** : Consultez la section « Références » en bas des articles Wikipédia pour trouver les sources primaires.

### Fiabilité des sources (Les LLM comme outils)

**Limitations critiques des LLM (ChatGPT, Claude, Gemini, Copilot, etc.) :**

* **Hallucinations** : Les LLM génèrent parfois des informations plausibles mais **inexactes** ou fictives, en particulier pour :
    * Les statistiques et chiffres précis (dates erronées, nombres inventés).
    * Les citations d'auteurs (les LLM peuvent inventer des citations).
    * Les sources et références (les LLM peuvent citer des articles qui n'existent pas).
* **Données de formation limitées** : La plupart des LLM ont une date limite de connaissance (ex: ChatGPT 3.5 s'arrête en 2021, GPT-4 en avril 2024). Ils ne connaissent pas les développements récents.
    * Ce [dépôt GitHub](https://github.com/HaoooWang/llm-knowledge-cutoff-dates) semble tenir à jour les données de formation de différents LLM.
* **Biais** : Les LLM reflètent les biais présents dans leurs données d'entraînement, ce qui peut favoriser certaines perspectives ou marginalisées d'autres.
* **Absence de traçabilité** : Impossible de vérifier les sources exactes utilisées pour générer une réponse.
* **Pas de révision par les pairs** : Contrairement aux articles scientifiques, les réponses des LLM ne sont pas vérifiées par des experts.

**Quand utiliser les LLM en recherche :**

* **Autorisé** :

    * Brainstorming et génération d'idées pour votre sujet.
    * Organisation et structuration de votre plan.
    * Clarification de concepts complexes (à vérifier ensuite auprès de sources fiables).
    * Reformulation ou résumé d'informations provenant de sources vérifiées.
    * Aide à la rédaction (orthographe, grammaire, clarté).

* **Interdit** :
    * Citer directement un LLM dans votre rapport sans vérification.
    * Utiliser les réponses des LLM comme preuves ou faits établis.
    * Présenter le contenu généré par un LLM comme votre propre travail (constitue du plagiat).
    * Remplacer la recherche académique par un LLM.

**Bonnes pratiques pour les LLM :**

* Utilisez les LLM pour **poser des questions de clarification**, puis vérifiez systématiquement les réponses auprès de sources académiques.
* Demandez au LLM de **citer ses sources** (les LLM peuvent le faire, mais n'oubliez pas : les citations peuvent être inexactes).
* Croisez les informations du LLM avec au moins **deux sources académiques indépendantes**.
* **Ne copiez jamais** une réponse de LLM directement dans votre rapport. Reformulez avec vos propres mots et vérifiez les faits.
* Indiquez si vous avez utilisé un LLM dans votre processus de recherche (certains établissements l'exigent dans une section "Méthodologie" ou "Outils utilisés").

## 3. Organiser l’information

### Cahier de recherche

Consignez vos recherches dans un journal structuré :

* Dressez la liste des recherches effectuées.
* Rédigez un court résumé pour chaque source afin de faciliter la mémorisation du contenu traité.

### Évaluation de la pertinence
* Relisez votre cahier de recherche pour valider la nécessité des informations trouvées.
* Utilisez une **pondération de 1 à 10** pour prioriser les éléments les plus significatifs.

### Ordonner et structurer
* Établissez un fil conducteur pour assurer la cohérence.
* Regroupez les sujets similaires ensemble en progressant vers les sujets distincts pour éviter les ruptures de flux (le "coq-à-l'âne").
* Le plan définitif doit correspondre à la table des matières finale du rapport.

## 4. Produire et présenter le rapport

### Rédaction (Le Fond)
* Assurez une cohérence constante avec le sujet principal.
* Pour les références, utilisez une norme reconnue (au choix : **APA** ou **IEEE**).

### Mise en forme (La Forme)
Structure technique type d'un rapport :

| Section | Éléments inclus | Pagination |
| :--- | :--- | :--- |
| **Présentation** | Page de titre (centrée verticalement) | Aucune |
| **Pré-contenu** | Remerciements, Table des matières, Liste des illustrations/tableaux | Chiffres romains |
| **Contenu** | Chapitres (Introduction, Développement, Conclusion) | Chiffres arabes |
| **Finale** | Médiagraphie, Annexes | Chiffres romains |

### Présentation écrite et orale
* **Oral** : Ne lisez pas le rapport. Vulgarisez le contenu et concentrez-vous sur l'essentiel en fonction du temps imparti.
* **Style écrit** :
    * Utilisez l'italique pour les mots étrangers.
    * Basez-vous sur des faits et référencez chaque affirmation.
    * **À bannir** : Langage familier, style parlé, et l'usage du "je" (privilégiez "on" ou "nous").
