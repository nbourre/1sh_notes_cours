# Présentation du stage

## Consignes

Vous devez préparer une courte présentation de **5 à 10 minutes**, sous forme de **table ronde**.

Vous n'avez pas besoin d'un support de présentation formel. L'objectif est de résumer votre expérience et vos apprentissages de manière claire et concise.

Cette consigne s'applique à deux situations :

* **avec stage** : vous présentez votre milieu de stage, votre expérience et vos apprentissages ;
* **sans stage** : vous présentez ce que vous avez fait pendant cette période en lien avec l'informatique, ainsi que les éléments d'intérêt que vous avez explorés.

## Parcours 1 : avec stage

### Introduction à votre présentation

Commencez par présenter brièvement :

* l'entreprise qui vous a accueilli ;
* votre département ou secteur de travail ;
* le poste occupé ;
* vos principales responsabilités.

### Points à aborder

Expliquez ensuite votre expérience de stage en traitant les éléments suivants.

#### Apport personnel

Présentez ce que le stage vous a apporté sur le plan personnel.

#### Apport professionnel

Présentez ce que le stage vous a apporté sur le plan professionnel.

#### Apprentissages réalisés

Répondez aux questions suivantes :

* Quels **apprentissages** avez-vous faits durant le stage ?
* Que pensiez-vous apprendre durant le stage ?
* Qu'auriez-vous aimé apprendre ?
* Dans quelle mesure ces objectifs ont-ils été atteints ? Pourquoi ?

#### Appréciation générale du stage

Par rapport à votre expérience **en général** :

* Quels aspects voudriez-vous améliorer ou modifier ?
* Quels aspects avez-vous appréciés ?
* Quels commentaires généraux feriez-vous sur votre stage ?

#### Après le diplôme

Abordez aussi vos **projets post-diplôme**.

## Parcours 2 : sans stage

Si vous n'avez pas effectué de stage, vous devez tout de même préparer une présentation structurée de **5 à 10 minutes**.

Dans ce cas, présentez ce que vous avez fait pendant cette période en lien avec l'informatique, par exemple :

* un projet personnel ;
* de l'apprentissage autonome ;
* des expérimentations techniques ;
* des recherches sur des domaines qui vous intéressent ;
* une exploration de carrières, d'outils ou de technologies.

Votre présentation peut notamment aborder les éléments suivants :

* les activités réalisées pendant cette période ;
* les sujets, technologies ou enjeux que vous avez explorés ;
* ce que vous avez appris sur le plan personnel et professionnel ;
* les difficultés rencontrées et la manière dont vous les avez surmontées ;
* les éléments que vous auriez aimé approfondir davantage ;
* ce que cette période vous a permis de mieux comprendre sur vos intérêts en informatique ;
* vos intentions ou projets pour la suite.

L'objectif reste le même : présenter une démarche structurée, montrer ce que cette période vous a apporté et établir des liens concrets avec votre parcours en informatique.

## Évaluation

L'évaluation est sur **13 points**.

| Critère | Points |
| :--- | ---: |
| Description du milieu de stage : entreprise | 3 |
| Description du milieu de stage : tâches | 3 |
| Apport personnel | 2 |
| Apport professionnel | 2 |
| Apprentissages | 2 |
| Général | 1 |

## Rappel

La présentation doit être concise, structurée et centrée sur votre expérience réelle, que celle-ci provienne d'un stage ou d'activités réalisées en lien avec l'informatique pendant la même période.
