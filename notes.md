# Expressions intéressantes : "vibe coding"

- Un bon développeur teste. Un mauvais développeur espère.
- "Ça fonctionne" ≠ "C’est bon"
- Le *vibe coding* permet de transformer une idée en code via des prompts.
- Tester, c’est vérifier, pas deviner.
- Un bon dev sait dire :  "Ce n’est pas prêt"
- Le vibe coding est une boucle de collaboration où l’humain garde le jugement.
